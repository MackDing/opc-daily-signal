⚠️ 反向检验：如果客户问"能不能让 Bot 自动回答订单状态"而你只能接 FAQ，说明你赌的是"装饰性需求"而不是"功能性需求"——前者支付意愿低且会被 Wix/Squarespace 一键插件吃掉。装饰性 SKU 必须做到 30 分钟交付，否则毛利归零。

## ⚡ 时间 ｜ Claude Code + 纯 HTML 的"不合理有效性"

→ HN 第 4 名（407 分，234 评论）：在 Claude Code 里用纯 HTML（不上 React/Next/Tailwind 工具链）做内部工具与 demo，速度提升一个数量级，因为 LLM 不需要在 framework 怪癖上消耗 tokens 和你自己的回路。一次写一个 file，refresh 就能看见结果。
🔑 动作：把你下一个 MVP / 客户演示页 / 内部 dashboard 的脚手架从"Next.js + shadcn"切换到"单文件 HTML + Alpine/HTMX + Tailwind CDN"。今天选一个 in-flight 的任务跑这个对照实验——用秒表测两次"从 prompt 到可点击 demo"的耗时。
🔮 二阶（3-6 个月）：当 LLM 编辑速度成为新瓶颈，"工具链复杂度"从过去的工程美德变成单人毛利的最大税。一人公司的技术栈会反向收敛到 1995 年（HTML + 一点 JS + 服务端渲染）。早转的人时薪先翻倍。
⚠️ 反向检验：如果项目需要离线/PWA、复杂状态机、或团队 ≥3 人协作，纯 HTML 会在第二周开始反噬。判断线：单页面 ≤ 800 行、无登录态共享、生命周期 < 6 个月——满足才上。
🔗 https://news.ycombinator.com/item?id=48071940

## 💀 风险 ｜ "委托给 LLM 会悄悄腐蚀你的文档" — 一人公司的隐性负债

→ arXiv 论文 HN 339 分：当用户把文档编辑任务委托给 LLM agent，原文中 7-12% 的事实陈述会在多轮编辑后被静默改写（数字、引用、人名、否定词翻转），且作者复核漏检率 > 60%。这不是"幻觉"，是**编辑路径上的熵积累**。
🔑 动作：今晚给所有"agent 可写"的文档目录加一道 git pre-commit hook：对比关键字段（金额、日期、客户名、合同条款）的 diff，红色高亮强制人工 ✓ 才能 commit。SKU 报价单、客户邮件草稿、技术合同——这三类必须落地。
🔮 二阶（6-18 个月）：随着一人公司把更多文书外包给 agent，第一波"agent 引发的合同纠纷 / 报价错发 / 客户失信"会出现在 2027。提前建立"diff-验证-签字"流程的人，将以"合规一人公司"差异化定价（+15-30%）。
⚠️ 反向检验：如果你跑 hook 一周后发现 0 次警报，要么文档不重要（删了省事），要么 hook 关键字段选错了（没覆盖到真实风险点）——后者比前者危险十倍。每月手动抽查 3 份做对照。
🔗 https://arxiv.org/abs/2604.15597
