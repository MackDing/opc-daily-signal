## 💰 单价 — "Skills 包"成为新产品形态

**事实**
GitHub Trending 本周榜单上同时出现两个非代码、非框架、非模型的新物种：
- `mattpocock/skills`：17,059 stars / 周
- `addyosmani/agent-skills`：9,198 stars / 周
两者都是把作者本地 `.claude/` 目录下的 SKILL.md（Anthropic 推出的 agent procedural memory 格式）整理后开源。star 速度超过同周大多数 framework。

**底层逻辑（为什么 OPC 要关注）**
过去一年 indie hacker 的可售品类是：SaaS、模板、Notion 模板、prompt 包。Skills pack 是第五种——它是「prompt + 流程 + 工具调用约定 + reference 文档」的封装。和 prompt 包的区别在于：skill 直接被 agent 调用执行，所见即所得，不需要用户复制粘贴；和 SaaS 的区别在于：它是 markdown 文件，零运维成本。
对 OPC：单价上限被打开（可以做 $19-$99 的"垂直行业 pack"），数量端有 GitHub/Anthropic marketplace 当分发，时间投入是把已有 workflow 写成 markdown，几乎为 0 增量成本。这是「把存量经验直接资产化」的稀有窗口。

**为什么不是噪音**
免费 repo 已经按"产品"在被消费——issues 区像在提需求、PR 像在贡献模块。这意味着付费意愿存在。

**反向检验**
- 如果 14 天后你的 repo star < 200 且 issue 区无人提需求，说明你的 skill 不是别人的痛点，而是你的私人 workflow 自嗨——别打磨 README，立刻换垂直。
- 如果免费版下载量很高但付费转化 < 1%，说明你没把"专家知识/私域答疑"打包进付费版——免费/付费的差距必须是「时间」差距，不是「文件数」差距。

**连锁效应（时间维度）**
- T+30 天：中文圈跟进，B站/小红书出现"我靠卖 Claude skill 月入 X"的视频
- T+90 天：首批 solo 案例（Gumroad/Sponsors 月入 $5k 级）公开
- T+180 天：Anthropic 官方 marketplace 上线，抽 15-30%；先发者吃留存，新进者要靠付费投流

**今天能改变的动作**
1. 列出你最熟的 3-5 个 SKILL.md
2. 选 1 个垂直行业（建议中文场景：律师函/小红书爆款/电商 listing/独立开发者税务）
3. 本周内 push 公开 repo + 挂 Gumroad $19 进阶版（templates+scripts+私域答疑）

🔗 https://github.com/mattpocock/skills
🔗 https://github.com/addyosmani/agent-skills

## ⚡ 时间 — Anthropic 亲自下场，垂直 Skill pack 的脚手架被开源

**事实**
`anthropics/financial-services` 本周 12,529 stars。Anthropic 官方仓库把"金融行业 skills"的目录结构、reference 文档、prompt 模板全部开源。

**底层逻辑**
官方下场做垂直 pack，等于把"格式标准 + 最佳实践骨架"免费送给市场。任何想做"X 行业 pack"的人不再需要从 0 设计目录、不再需要猜 reference 该放什么——直接 fork、替换内容、推 v0.1。这是把 1 个月的产品打磨压缩到 1 周的脚手架。

**对 OPC 的意义**
时间端直接砍 70%。原本你要花 2-3 周思考"金融 pack 该长什么样"，现在 48 小时可以推出 v0.1。窗口期是稀缺的——一旦中文圈大量 fork 出现，标准会同质化，先发者拿走 SEO 和 GitHub trending。

**反向检验**
- 如果你 fork 后只是改 README 没新增 reference 文档与 prompt 例子——star 会停在 50 以下。纯目录搬运没有 OPC 单价，必须注入 1 个外部人难复刻的「领域知识/数据/案例库」。
- 如果你不熟该行业，硬 fork 也不行——领域 know-how 的厚度是护城河，没有就别做。

**连锁效应（时间维度）**
- T+14 天：第一批中文 fork（教育/电商/法律）出现
- T+30 天：5+ 中文 fork 拿到 1k+ star
- T+60-90 天：OpenAI/Google 跟进官方 skill 仓，标准开始统一；窗口期关闭

**今天能改变的动作**
今晚 fork → 选你最熟的中文行业 → 替换内容 → 48 小时 v0.1 → 配一篇中文博客「我把 Anthropic 金融模板改成 X 行业」做引流。

## 📈 数量 — Codex 进 ChatGPT mobile，开发者工具的用户基数阶跃式扩容

**事实**
OpenAI 官方宣布 Codex 进入 ChatGPT 移动 app：手机上可以拉远程任务、查看 PR、合并代码。

**底层逻辑**
ChatGPT 移动端 MAU 1 亿+，绝大多数不是开发者。Codex 嵌入移动端意味着两件事：
1. 现有开发者的「可用编码时段」从 8h 工位时间延伸到 24h 任意场景（地铁、咖啡馆、马桶上）
2. 大量「半开发者」（PM、运营、设计师、学生）第一次有了「在手机上 ship 代码」的低门槛入口
对 OPC：数量端阶跃。任何能在「5 分钟一句话触发」场景下闭环的工具，潜在用户池从「Cursor 用户」扩到「ChatGPT mobile 用户」，约 50-100 倍。

**反向检验**
- 如果你的产品需要本地文件系统/长 session/复杂 IDE 状态，移动端流量来了也接不住——别强行做 mobile 适配，反向操作：把你的工具变成 codex 可调用的 hosted service，让 codex 当客户。
- 如果你的产品没有 GitHub App 集成，没有 webhook，移动用户根本触发不到你——这是基础门槛。

**连锁效应（时间维度）**
- T+6 周：mobile-first 的轻量 dev 工具（codereview bot、cron 助手、文档生成器）DAU 翻倍
- T+3 个月：出现"只在手机上写代码"的 indie 开发者画像
- T+6 个月：IDE 厂商被倒逼做 mobile companion app

**今天能改变的动作**
1. 给你的 SaaS 加一条 GitHub App + webhook 集成（10 行代码），支持 `@codex` 触发
2. 录一条「通勤 5 分钟修 bug」短视频发 X / 小红书 / B站
3. 把 onboarding 文档的第一个 demo 改成「在手机上完成」的版本

🔗 https://openai.com/index/work-with-codex-from-anywhere/

## 💀 成本 — arXiv 对"AI 幻觉引用"封号 1 年，平台开始反噬

**事实**
arXiv 出新规：论文若被发现含幻觉/伪造的参考文献，作者封禁 1 年。这是顶级学术平台第一次把"AI 输出未核对"列为可封号违规。

**底层逻辑**
过去 18 个月「AI 批量产内容」是 OPC 的数量倍增器。现在平台开始反向收紧：每一条 AI 内容都自带「真实性责任」。对 OPC，这是成本端的隐性翻倍——你以为节省了 1 小时写作时间，但漏掉一条幻觉引用的潜在代价是封号、信任崩塌、客户合同终止。

**为什么会扩散**
arXiv 是学术平台的标杆。它一旦立规矩，SSRN、出版社、知乎、B站、小红书的相关合规团队会有 talking point 跟进。这不是「学术界的事」，是所有「平台 vs UGC」博弈的转折信号。

**反向检验（条件句）**
- 如果你觉得"我又不发论文，跟我无关"——错。任何对外署名内容（公众号、newsletter、客户报告、提案）都受同一类标准约束。
- 如果你的内容工厂没有"事实核对"环节，只有"AI 生成 → 排版 → 发布"——这条流水线在 6 个月内会成为负债，不是资产。
- 如果你做 SaaS 给企业客户输出 AI 内容——客户合同里很快会要求「真实性条款」，没流程的乙方被淘汰。

**连锁效应（时间维度）**
- T+30 天：SSRN 等学术平台跟进
- T+60 天：至少 1 个中文平台（知乎/小红书/B站）出类似规则，AI 批量号被限流
- T+6 个月：SaaS 服务合同条款里出现「AI 真实性」相关条款

**今天能改变的动作**
1. 在你所有对外内容的发布 SOP 里加一条「引用/数据/人名/链接 必须人工 grep 核对」checklist
2. 写一段 lint 脚本，对每篇文章自动列出所有数字+引用，生成核对表
3. 客户交付物前置「真实性核对」环节并在合同里背书——把成本变成单价
