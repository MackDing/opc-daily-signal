### 一个 CLAUDE.md 文件 = 72K Stars：把经验打包成配置文件是新产品形态

forrestchang/andrej-karpathy-skills 本周狂飙 40,732 stars（总计 72K），仅靠一个 CLAUDE.md 文件——把 Andrej Karpathy 对 LLM 编程陷阱的观察打包成 Claude Code 的行为约束。没有 SaaS，没有后端，没有订阅——一个 markdown 文件。

**底层逻辑：** 这不是开源项目的传统路径。它验证了一个新品类：**"AI 行为配置即产品"**。当所有人都在卷 AI 应用层，真正稀缺的是"让 AI 按你的方式工作"的专家经验。这个经验的载体不必是 SaaS，可以是一份配置文件、一个 system prompt、一套 rules。

**为什么 OPC 要特别关注：** 大公司做 AI 工具，OPC 做 AI 工具的"使用说明书"。这是典型的寄生定位——不跟平台竞争，在平台生态里卖铲子。单价天花板取决于你打包的经验有多稀缺。Karpathy 的名字带来了初始流量，但 72K stars 说明内容本身有独立价值。

**🔑 可复制动作：** 选你最熟悉的 AI 工具（Claude Code、Cursor、Copilot），写一份基于你实战踩坑的 rules/CLAUDE.md，开源发布。如果它能帮人少走弯路，就能变成付费咨询/定制服务的入口。

**逆向检验：** 如果你打包的经验是"通用最佳实践"（Google 一搜就有），没人会 star。只有当经验来自特定领域（如"用 Claude Code 写金融合规代码的 14 条规则"），才有提价空间。

### VidStudio：浏览器视频编辑器 Show HN 拿 238 分，SEO 长尾页面策略值得抄

VidStudio 是一个纯浏览器端视频编辑器（FFmpeg WASM，文件不上传），Show HN 获 238 分。产品本身不新，但它的增长策略值得拆解：首页包含 30+ 个针对具体平台的落地页——"Compress for Discord""Resize for TikTok""Compress for WhatsApp"——每个都是独立的 SEO 入口。

**底层逻辑：** 一个核心功能（视频处理）× 30 个使用场景页 = 30 个独立的搜索入口。用户不会搜"browser video editor"，会搜"compress video for discord free"。每个落地页精准匹配一个长尾搜索意图，成本几乎为零（只是同一功能的不同参数预设）。

**为什么 OPC 要特别关注：** 这是"一次构建，多次分发"的教科书案例。OPC 最稀缺的是时间，SEO 长尾页面策略的 ROI 是累积的——每多一个页面，搜索曝光量线性增长，但维护成本几乎不变。

**🔑 可复制动作：** 审视你现有产品/工具的核心功能，列出 10 个用户会搜索的具体场景（"[动作] + [平台] + free/online"），为每个场景创建独立落地页。模板化生成，30 分钟搞定。

### AI 编程工具集体缩水：GitHub Copilot 暂停注册 + Claude Code 可能移出 Pro

GitHub 官方博客（4月20日发布，300 分 HN）宣布三项变更：

2. **收紧用量限制**——Pro+ 限额是 Pro 的 5 倍以上，Pro 用户被鼓励升级
3. **Pro 档移除 Opus 模型**——Opus 4.7 仅 Pro+ 可用，Opus 4.5/4.6 将从 Pro+ 移除

**底层逻辑：** Agentic workflows 的算力消耗远超原有定价模型设计——长时间、并行化的 session 让 $10-20/月的订阅完全 hold 不住。这不是涨价，是"不可能三角"的必然坍缩：便宜 × 强模型 × 无限用量，只能选两个。

**为什么 OPC 要特别关注：** AI 编程工具是 OPC 最大的杠杆倍增器。当供应商在调整规则时，没有 Plan B 的 OPC 是最脆弱的。现在是建立本地/开源替代栈的窗口期：

- andrej-karpathy-skills 的 CLAUDE.md 能让现有配额产出更高质量代码

**逆向检验：** 如果你的 Copilot 用量从未接近限额（大多数用户确实如此），这个变更对你没有实际影响。但如果你是重度 agentic 用户（每天跑长 session），影响是立竿见影的。

## 💀 死亡信号（别人踩的坑）

### Vercel OAuth 供应链攻击：环境变量泄露，部署在 Vercel 的立刻检查

Trend Micro 披露 Vercel 平台遭 OAuth 供应链攻击（HN 253 分），攻击者通过 OAuth 流程获取平台环境变量。这意味着你存在 Vercel 环境变量里的 API keys、数据库密码、第三方服务凭证可能已暴露。

**死因一句话：** 平台级 OAuth 实现漏洞 → 环境变量被窃取 → 所有部署在该平台上的应用的密钥链全部暴露。

**为什么 OPC 要特别关注：** OPC 大量使用 Vercel/Netlify 这类平台部署——因为省时间。但"省时间"的代价是把所有鸡蛋放在一个篮子里。大公司有安全团队轮换密钥，OPC 通常一套 key 用到死。一次泄露 = 所有服务同时沦陷。

**🔑 你该检查的一件事：** 今天登录 Vercel dashboard，检查所有环境变量。轮换（rotate）所有 API keys 和数据库密码。未来：不要在环境变量中存放长期有效的凭证，改用短期 token 或 secret manager。

**连锁效应：** 这个信号和上面的 AI 工具缩水信号叠加——当你同时依赖 Vercel 部署 + Copilot 编程 + OpenAI API，任何一个环节出问题都会让你的整个工作流瘫痪。OPC 需要的不是"最好的工具"，而是"最有韧性的工具组合"。

🔗 https://www.trendmicro.com/en_us/research/26/d/vercel-breach-oauth-supply-chain.html
