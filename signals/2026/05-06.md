# OPC 日报 · 2026-05-06

> 第一性原理：OPC存活 = (单价 × 数量) / 投入时间 > 成本

---

## 💰 单价 | Vision Agent 比结构化 API 贵 45 倍 — AI 自动化产品的定价基线被刷新

**→ 事实**
Reflex 团队把同一个 admin panel 用两种 agent 跑相同任务（找 Smith 客户最近的 pending 订单 + 接受 reviews + 标记交付）。Vision agent（browser-use 0.12 + Claude Sonnet 驱屏截图点击）跑了 53 步 / 551k tokens，且**首次未能完成任务**；API agent（同模型，调用底层 event handler）只用 **8 次调用 / 12k tokens** 完成。成本差 ~45 倍，准确率差更大。

**🔑 动作**
若你正在做"AI 帮我用某个 SaaS"类产品（HN 现在每天 3-5 个 Show HN 是这个方向），**今天就停止 vision/browser-use 路线，改做 MCP/REST 薄壳**。即使被自动化的目标软件没 API，自己写一层 200 行的 scraper-to-API 适配层比 vision agent 长期 token 成本便宜两个数量级，且可以稳定向客户报价（每任务 $0.02 vs $0.90）。

**🔮 二阶效应（3-6 个月）**
当 vision agent 的真实 token 账单出现在客户发票上，会出现一波"AI 自动化"产品的退订潮，定价模型从 per-seat 转向 per-task。**反向机会**：做"为某个垂直 SaaS 自动生成 MCP 适配层"的工具，本身是一个 OPC 可承接的小生意（参考 Reflex 自己就是这条路）。

**⚠️ 反向检验**
如果你的目标客户**就是为了 audit trail 才要 vision**（金融/医疗合规场景下截图就是凭证），那 45x 成本是合规溢价不是浪费，此时 API 路线反而失败。判断条件：客户合同里是否写明"操作过程需可视化留证"。

**🔗 来源**: https://reflex.dev/blog/computer-use-is-45x-more-expensive-than-structured-apis/

## ⚡ 投入时间 | "Skills" 成为本周 GitHub 第二爆款（mattpocock/skills 单周 +25k stars）

**→ 事实**
本周 GitHub Trending 出现明显信号：mattpocock/skills（"Skills for Real Engineers, straight from my .claude directory"）单周 25,389 stars，ComposioHQ/awesome-codex-skills 3,370 stars。Skills = 可复用的 markdown 工作流定义（Anthropic Claude Code / OpenAI Codex 都已原生支持），相当于把"个人最佳实践"打包成 agent 可调用的程序。Warp（agentic terminal）同期 28k stars 也属于同一波。

**🔑 动作**
今晚把你 7 天内用 Claude/Codex 做过 5 次以上的任务（发周报、改 PR 描述、清 inbox、生成 diagram、写 SQL）每个写成一个 SKILL.md（YAML frontmatter + 步骤），放到自己 GitHub repo。一周内你就有 5-10 个 skills，**下周做同类任务的耗时砍 60-80%**——这是直接降低 OPC 公式分母（投入时间）。

**🔮 二阶效应（1-3 个月）**
"个人 skills 库"会像 dotfiles 一样变成开发者人格名片，公开 repo 会带来咨询/合作流量。**6 个月内出现"skills marketplace"**（付费 skills、企业 skills 审计），先发者占位 SEO + GitHub stars 双重红利。mattpocock 的崛起路径（教 TypeScript → 出 skills 库 → 卖课）是一个复刻模板。

**⚠️ 反向检验**
如果你的工作流里 **<3 个任务有重复结构**（每天都做不同的事），写 skills 是过度工程，不如直接 prompt。判断条件：列出本周做过的事，看是否有 ≥3 类任务出现 3+ 次。

**🔗 来源**: https://github.com/mattpocock/skills

## 📈 数量 | "产品 tour 几乎被秒关"——Frigade 联创给出唯一有效激活模式

**→ 事实**
Frigade 联创 Eric Brownrout 拆解 onboarding 数据：大多数用户在 product tour 的第 1 步就 dismiss，原因是 tour 假设用户想"被介绍产品"，但用户上来想**完成一个具体任务**。唯一持续提升 activation 的模式是"contextual checklist"——把首次成功路径拆成 3-5 个具体待办，挂在主界面右下角，用户每完成一项打勾。Notion/Linear/Stripe/Cal/Loom 都用此模式。

**🔑 动作**
本周删掉你产品里的 tooltip 引导 / modal welcome tour，换成 3 步 checklist：步骤 1 必须是"用户第一次产生价值"（生成第一份报告 / 发出第一封邮件 / 接到第一个 webhook），步骤 2-3 是 retention hook。**不要用步骤 1 收集 profile 信息**——这是最大杀手。

**🔮 二阶效应（2-4 周）**
D1 retention 通常能从 ~25% 提升到 35-45%（Frigade 公开 case study 区间），等于在不增加流量、不涨价的情况下把"数量"变量提升 40%——这是 OPC 公式分子里最便宜的 lever。**长期**：当 LLM 接管"问产品怎么用"，product tour 整个品类会消失，但 onboarding checklist 会演化成"agent 替我跑完前 3 步"。

**⚠️ 反向检验**
如果你的产品是 **single-action utility**（一个按钮 = 一次价值，例如 image compressor），上 checklist 反而是噪音。判断条件：用户首次价值是否需要 ≥2 步配置。否则直接默认值 + 一个 CTA。

**🔗 来源**: https://productonboarding.com/articles/why-product-tours-get-skipped

## 💀 成本 | Coinbase 再裁 14%——加密巨头都撑不住的人头成本，反证 OPC 结构红利

**→ 事实**
Brian Armstrong 公开宣布裁员 14%（约 1000+ 人），距上一轮裁员仅 18 个月。Coinbase 在 2026 年 BTC 周期高位、营收同比正增长的情况下仍砍人，理由是"组织复杂度上升、决策速度下降"。HN 327 条评论里高频信号：员工反馈"会议占 60% 时间"、"产品 review 流程 6 周"。**这不是失败叙事，是一个数学结构**：每多 1 名全职员工 → 协调成本以 O(n²) 上升 → 边际产出在某个阈值后转负。

**🔑 动作**
把这条钉在墙上：**不要为了"看起来正规"而招第二个全职**。优先级：自动化 > 外包按项目 > 兼职合作 > 全职。今天审视你的 todo list，标出"我做太慢但请人不值"的任务，下周用 skills + agent 砍掉一项；如果你已经 2 人，问"如果我们俩各自再快 30%，是否就不需要第 3 个 hire？"——答案 80% 是"不需要"。

**🔮 二阶效应（6-12 个月）**
2026 下半年大厂裁员 + AI agent 价格下降，会释放一批"前 Senior PM/Eng + 自带 AI 工具栈"的独立 builder，**B2B SaaS 的客户群中"1-3 人公司"占比会从今天的 ~15% 涨到 25-30%**。反向机会：现在做"为 1 人公司设计"的财务/合规/客服工具（不是把 enterprise SaaS 缩小，而是从零设计），未来 12 个月这块需求曲线最陡。

**⚠️ 反向检验**
如果你做的是**强网络效应产品**（marketplace、社交、协议），1 人公司不可能赢，必须烧人头抢供给两端。判断条件：你的 retention 是否依赖"另一群用户也在场"。否则 solo 路线成立。

**🔗 来源**: https://twitter.com/brian_armstrong/status/2051616759145185723

- Reflex 的 vision-vs-API 成本差与 Anthropic 同日发布的 finance/insurance agents（HN #48023533）形成对照：Anthropic 也走 structured tool-use 路线，未押 vision——头部信号一致。
- Skills 趋势 + Warp 28k stars + ruflo 9k stars 三个独立信号同向：agent 工作流的"打包格式"正在标准化。
- Coinbase 裁员 + Xbox 终结 Copilot AI 开发（HN #48029753）+ Meta 版权诉讼（#48026207）三条同日大厂逆风新闻，反衬独立 builder 的结构稳定性。

## 连锁效应

Vision-agent 失败 → token 成本反弹 → AI SaaS 重新走 per-task 定价 → 单价模型回归"做一件事赚一份钱"，正好是 OPC 公式最适合的形态。Skills 标准化 → 个人工作流可移植 → 独立开发者的"换工具迁移成本"降低 → AI 工具市场进入价格战 → OPC 成本进一步下降。两条线交汇于 2026Q3-Q4：**届时是个人产品定价 + 工具组合的最佳重排时机**。
